# Development workflow

Plan first. Before implementing any feature or making changes, create an outline and plan. Verify it for correctness, privacy, and security to avoid regressions.

# Commit style

When generating commit messages, always use a short title line followed by a bulleted list body. Keep the title under 50 characters. Use conventional commit prefixes (e.g. `module: verb`). Follow the repo's existing commit style — check `git log --oneline -10` for reference.
